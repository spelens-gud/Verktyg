# README 文档编写规范

本规范定义了为 Go 组件编写 README.md 文档的标准格式和内容要求。

## 文档语言

- 所有 README 文档必须使用**中文**编写
- 代码示例中的注释使用中文
- 变量名、函数名等保持英文（遵循 Go 语言规范）

## 文档结构

### 1. 标题和简介

```markdown
# 包名 - 简短描述

`包名` 包的一句话功能描述，说明其核心价值。
```

**要求：**
- 使用一级标题，格式：`# 包名 - 简短描述`
- 简介控制在 1-2 句话内
- 突出核心功能和使用场景

### 2. 核心功能

使用二级标题 `## 核心功能`，按功能模块组织：

```markdown
## 核心功能

### 1. 功能模块名称

功能描述段落。

**关键特性：**
- 特性1
- 特性2

**主要函数/接口：**
\```go
// 代码示例
\```
```

**要求：**
- 每个功能模块使用三级标题
- 先描述功能，再列举特性
- 提供简洁的代码示例
- 代码示例必须可运行或清晰展示用法

### 3. 接口定义（如适用）

对于接口包，展示核心接口定义：

```markdown
### 接口名称

\```go
type InterfaceName interface {
    Method1() error
    Method2(param Type) (result Type, err error)
}
\```

**方法说明：**
- `Method1` - 方法功能描述
- `Method2` - 方法功能描述
```

### 4. 配置说明（如适用）

如果组件涉及配置，使用表格展示：

```markdown
## 配置项

### 环境变量

| 环境变量 | 说明 | 默认值 | 必填 |
|---------|------|--------|------|
| `VAR_NAME` | 变量说明 | default | 是/否 |
```

**要求：**
- 使用 Markdown 表格格式
- 环境变量名使用代码格式（反引号）
- 提供清晰的默认值
- 标注是否必填

### 5. 使用示例

```markdown
## 使用示例

### 基本使用

\```go
package main

import (
    "fmt"
    "包路径"
)

func main() {
    // 示例代码
}
\```

### 高级用法

\```go
// 更复杂的示例
\```
```

**要求：**
- 至少提供 2-3 个示例
- 从简单到复杂递进
- 示例代码必须完整可运行
- 包含必要的导入语句
- 添加关键步骤的中文注释

### 6. 设计原则

```markdown
## 设计原则

1. **原则名称** - 原则说明
2. **原则名称** - 原则说明
```

**要求：**
- 列出 3-5 个核心设计原则
- 使用粗体突出原则名称
- 简洁说明每个原则的含义

### 7. 相关实现/依赖

```markdown
## 相关实现

- `包路径` - 功能说明
- `包路径` - 功能说明

## 依赖项

- `依赖包` - 用途说明
```

### 8. 注意事项

```markdown
## 注意事项

1. 注意事项1的详细说明
2. 注意事项2的详细说明
3. 常见陷阱和最佳实践
```

**要求：**
- 列出使用时的重要注意点
- 包含常见错误和解决方案
- 提供最佳实践建议

## 格式规范

### 代码块

- 使用三个反引号标记代码块
- 指定语言类型：\```go
- 代码缩进使用 Tab 或 4 个空格
- 保持代码简洁，避免冗余

### 列表

- 无序列表使用 `-` 或 `*`
- 有序列表使用数字 `1.`, `2.`
- 列表项之间保持一致的格式

### 强调

- 使用 `**粗体**` 强调重要概念
- 使用 `` `代码格式` `` 标记代码、变量名、函数名
- 避免过度使用强调

### 链接

- 内部链接：`[文本](#锚点)`
- 外部链接：`[文本](URL)`
- 相对路径：`[文件](../path/to/file)`

## 内容原则

### 完整性

- 覆盖所有主要功能
- 包含足够的示例代码
- 提供必要的配置说明
- 列出相关依赖和实现

### 准确性

- 代码示例必须经过测试
- 配置项说明准确无误
- 版本信息保持更新
- 避免误导性描述

### 可读性

- 使用清晰的标题层次
- 段落简洁明了
- 适当使用表格和列表
- 代码示例易于理解

### 实用性

- 提供实际可用的示例
- 包含常见使用场景
- 解决实际问题
- 便于快速上手

## 特殊场景

### 接口包（interfaces/）

- 重点展示接口定义
- 说明接口的设计意图
- 列出实现该接口的包
- 提供接口使用示例

### 实现包（implements/）

- 说明实现了哪些接口
- 展示初始化和配置方法
- 提供完整的使用流程
- 包含错误处理示例

### 工具包（kits/）

- 列出所有工具函数
- 按功能分组说明
- 提供典型使用场景
- 强调线程安全性等特性

## 维护要求

- 代码变更时同步更新文档
- 定期检查示例代码的有效性
- 及时补充新功能的说明
- 保持文档与代码的一致性

## 质量检查清单

在完成 README 文档后，检查以下项目：

- [ ] 标题和简介清晰明了
- [ ] 核心功能完整覆盖
- [ ] 代码示例可运行
- [ ] 配置说明准确完整
- [ ] 使用示例由浅入深
- [ ] 设计原则清晰表达
- [ ] 注意事项实用有效
- [ ] 格式规范统一
- [ ] 中文表达流畅
- [ ] 无错别字和语法错误
